# Performance Benchmark — v3.9.0

> Introduces **browser / worker mode** — slothlet can compose and run an api tree without a filesystem or Node built-ins by consuming a pre-generated manifest and loading each leaf through a resolvable module specifier. In Node mode the existing directory-scan path is unchanged, so Node-mode startup/call perf is unaffected. Benchmark date: June 2026 — re-benchmarked from the `v3.9.0` tag on Node.js 24 with `--conditions=slothlet-dev`; numbers reflect this version's code on current hardware, not its original release environment.

## Summary

| Metric                                | Eager   | Lazy     | Winner | Delta          |
| ------------------------------------- | ------- | -------- | ------ | -------------- |
| **Startup** (10-iter avg)             | 22.47ms | 22.38ms  | Lazy   | ~0.1ms (noise) |
| **Function Calls** (100-iter avg)     | 8.37μs  | 4.85μs   | Lazy   | ~3.5μs (noise) |
| **Materialization** (lazy first call) | —       | 350.01μs | —      | one-time       |

## Notes

- Browser mode is a separate load path (manifest + specifier resolution) measured in the browser smoke suite, not this Node benchmark; Node-mode steady-state perf tracks v3.8.0. The wider eager-call figure here is within the benchmark's known run-to-run spread (eager call max routinely exceeds 100μs).
- Micro-benchmark caveat: run-to-run variance is high (startup spreads tens of ms; calls a few μs), so eager-vs-lazy deltas within a few ms / sub-μs are measurement noise, not real differences.
- Re-benchmarked from the release tag on current hardware/Node.js 24; original release-time numbers are not reproduced here.
