# Performance Benchmark — v3.10.0

> Adds **synthetic / in-memory leaves** for `api.slothlet.api.add()` and **integrates the hook system with the permission system**. Both are additive: synthetic leaves flow through the same flatten/wrap pipeline as file-backed leaves, and hook permission gating is inert unless a `permissions` block is configured — so the steady-state startup and call paths are unchanged. Benchmark date: June 2026, Node.js 24 with `--conditions=slothlet-dev` (current release).

## Summary

| Metric                                | Eager   | Lazy     | Winner | Delta          |
| ------------------------------------- | ------- | -------- | ------ | -------------- |
| **Startup** (10-iter avg)             | 21.35ms | 21.18ms  | Lazy   | ~0.2ms (noise) |
| **Function Calls** (100-iter avg)     | 4.63μs  | 5.00μs   | Eager  | ~0.4μs (noise) |
| **Materialization** (lazy first call) | —       | 340.89μs | —      | one-time       |

## Raw Numbers

```text
Eager startup:  21.35ms
Lazy startup:   21.18ms
Eager calls:     4.63μs
Lazy calls:      5.00μs
Lazy first call (materialization): 340.89μs
```

## Notes

- Synthetic-leaf mounting reuses the existing flatten/wrap pipeline (no new per-call cost), and hook gating only runs when permissions are configured — so default-configuration perf is in line with v3.9.x.
- Micro-benchmark caveat: run-to-run variance is high (startup spreads tens of ms; calls a few μs), so eager-vs-lazy deltas within a few ms / sub-μs are measurement noise, not real differences.
- Benchmarked from the current release working tree on Node.js 24.
