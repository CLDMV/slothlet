# Performance Benchmark — v3.8.0

> Adds the **module discovery + mount pipeline** (`api.slothlet.api.modules.*`) for composing subsystems shipped as separate npm packages into the api tree at runtime. Discovery and mounting are compose-time operations (walk the filesystem, validate manifests, graft via `api.add()`); they do not touch the steady-state call path. Benchmark date: June 2026 — re-benchmarked from the `v3.8.0` tag on Node.js 24 with `--conditions=slothlet-dev`; numbers reflect this version's code on current hardware, not its original release environment.

## Summary

| Metric                                | Eager   | Lazy     | Winner | Delta          |
| ------------------------------------- | ------- | -------- | ------ | -------------- |
| **Startup** (10-iter avg)             | 22.35ms | 21.72ms  | Lazy   | ~0.6ms (noise) |
| **Function Calls** (100-iter avg)     | 6.07μs  | 5.44μs   | Lazy   | ~0.6μs (noise) |
| **Materialization** (lazy first call) | —       | 362.41μs | —      | one-time       |

## Raw Numbers

```text
Eager startup:  22.35ms

```

## Notes

- Module discovery/mount cost is paid once at compose time, not per call; the benchmark's steady-state numbers track v3.7.0.
- Micro-benchmark caveat: run-to-run variance is high (startup spreads tens of ms; calls a few μs), so eager-vs-lazy deltas within a few ms / sub-μs are measurement noise, not real differences.
- Re-benchmarked from the release tag on current hardware/Node.js 24; original release-time numbers are not reproduced here.
