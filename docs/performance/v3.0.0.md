# Performance Benchmark — v3.0.0

> Baseline release — first stable v3.  
> Benchmark date: April 14, 2026  
> Node.js runtime with `--conditions=slothlet-dev`

## Summary

| Metric | Eager | Lazy | Winner | Delta |
|--------|-------|------|--------|-------|
| **Startup** | 48.87ms | 35.36ms | Lazy | 1.4x faster |
| **Function Calls** | 9.29μs | 13.70μs | Eager | 1.5x faster |
| **Materialization** | — | 947μs | — | one-time |

## Startup Time (10 iterations)

```
Eager Mode:  48.87ms (avg) | 33.60ms (min) | 130.67ms (max)
Lazy Mode:   35.36ms (avg) | 28.98ms (min) |  44.12ms (max)
```

## Function Call Performance (100 iterations)

```
Eager Calls:          9.29μs (avg) |  4.82μs (min) |  62.63μs (max)
Lazy Subsequent:     13.70μs (avg) |  8.40μs (min) |  71.39μs (max)
Lazy First Call:    946.91μs (materialization)
```

## Complex Module Access (100 iterations)

```
Eager Nested:    18.19μs (avg)
Lazy First:     127.94μs (materialization)
Lazy Subsequent: 11.14μs (avg)
```

## Per-Module Breakdown (50 iterations)

| Module | Eager (avg) | Lazy First | Lazy Sub (avg) |
|--------|-------------|------------|----------------|
| math | 10.56μs | 104.46μs | 8.29μs |
| string | 13.39μs | 90.61μs | 6.43μs |
| funcmod | 4.62μs | 55.28μs | 2.82μs |

## Notes

- This is the v3.0.0 baseline — no versioning, no permissions, no env snapshot.
- Eager calls win in the Test 2 average; lazy wins in per-module tests (Test 4). Single-digit μs differences are within V8 JIT noise.
