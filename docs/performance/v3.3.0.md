# Performance Benchmark — v3.3.0

> Added Permission System: PermissionManager handler, shared pattern matcher, applyTrap enforcement, audit events. Permission system is **off by default** — zero overhead when not configured. Benchmark date: April 14, 2026 Node.js runtime with `--conditions=slothlet-dev`

## Summary — Permissions Off (default)

| Metric              | Eager   | Lazy    | Winner | Delta       |
| ------------------- | ------- | ------- | ------ | ----------- |
| **Startup**         | 54.94ms | 40.21ms | Lazy   | 1.4x faster |
| **Function Calls**  | 16.37μs | 13.86μs | Lazy   | 1.2x faster |
| **Materialization** | —       | 1.02ms  | —      | one-time    |

## Summary — Permissions On (`permissions: { defaultPolicy: "allow" }`)

| Metric              | Eager   | Lazy    | Winner | Delta       |
| ------------------- | ------- | ------- | ------ | ----------- |
| **Startup**         | 55.38ms | 38.73ms | Lazy   | 1.4x faster |
| **Function Calls**  | 14.27μs | 11.74μs | Lazy   | 1.2x faster |
| **Materialization** | —       | 895μs   | —      | one-time    |

## Permissions Impact

| Metric        | Off     | On      | Difference     |
| ------------- | ------- | ------- | -------------- |
| Eager Startup | 54.94ms | 55.38ms | +0.4ms (noise) |
| Lazy Startup  | 40.21ms | 38.73ms | −1.5ms (noise) |
| Eager Calls   | 16.37μs | 14.27μs | −2.1μs (noise) |
| Lazy Calls    | 13.86μs | 11.74μs | −2.1μs (noise) |

**Conclusion:** Enabling permissions with `defaultPolicy: "allow"` and no rules adds zero measurable overhead. The `isEnabled()` guard short-circuits before any pattern matching when permissions are off. Even when enabled, the cost is within measurement noise because the benchmark modules make direct calls without rule evaluation (no caller context in the benchmark).

## Notes

- The permission system is off by default. When no `permissions` config is provided, `PermissionManager.isEnabled()` returns `false` and the `applyTrap` check short-circuits with zero work.
- The ~5ms increase in eager startup vs v3.2.3 is likely due to the additional handler registration (PermissionManager) and the heavier import graph. This is within normal run-to-run variance.
- Call-time differences between "off" and "on" are within noise (±2μs). The benchmark doesn't trigger rule evaluation since calls are made from the test harness (no caller wrapper context).
