# Performance Benchmark — v3.9.2

> Fixes a **critical async-runtime blow-up**: in `runtime: "async"`, chainable class instances were double-wrapped, producing `2^(N+4) − 1` Proxy layers for an N-step fluent chain (an 8-step chain took minutes). Wrapped Proxies now carry a sentinel and re-wrapping is idempotent, so fluent chains stay flat (one Proxy layer per step) and resolve in **linear** time. Benchmark date: June 2026 — re-benchmarked from the `v3.9.2` tag on Node.js 24 with `--conditions=slothlet-dev`; numbers reflect this version's code on current hardware, not its original release environment.

## Summary

## Notes

- The headline fix targets **fluent chains on chainable class instances in async runtime** — the aggregated benchmark below does not exercise deep chains, so its steady-state numbers track v3.9.0. The impact (exponential → linear) shows on query builders / fluent SDK clients; an 8-step chain went from minutes to milliseconds.
- Micro-benchmark caveat: run-to-run variance is high (startup spreads tens of ms; calls a few μs), so eager-vs-lazy deltas within a few ms / sub-μs are measurement noise, not real differences.
- Re-benchmarked from the release tag on current hardware/Node.js 24; original release-time numbers are not reproduced here.
