# Performance Benchmark — v3.5.0

> Ships TypeScript runtime-import fixes (bare specifiers like `@cldmv/slothlet/runtime` now resolve from `.ts`/`.mts`) and the `slothlet typegen` CLI. Both are build-time / module-load-path changes; the steady-state startup and call paths are unchanged. Benchmark date: June 2026 — re-benchmarked from the `v3.5.0` tag on Node.js 24 with `--conditions=slothlet-dev`; numbers reflect this version's code on current hardware, not its original release environment.

## Summary

| Metric                                | Eager   | Lazy     | Winner | Delta          |
| ------------------------------------- | ------- | -------- | ------ | -------------- |
| **Startup** (10-iter avg)             | 20.51ms | 20.34ms  | Lazy   | ~0.2ms (noise) |
| **Function Calls** (100-iter avg)     | 5.52μs  | 4.21μs   | Lazy   | ~1.3μs (noise) |
| **Materialization** (lazy first call) | —       | 311.06μs | —      | one-time       |

## Notes

- `typegen` runs on demand (e.g. from `predev`/`prebuild`) and the TS-import fix changes the module cache path, not the hot path — runtime perf tracks v3.4.0.
- Micro-benchmark caveat: run-to-run variance is high (startup spreads tens of ms; calls a few μs), so eager-vs-lazy deltas within a few ms / sub-μs are measurement noise, not real differences.
- Re-benchmarked from the release tag on current hardware/Node.js 24; original release-time numbers are not reproduced here.
