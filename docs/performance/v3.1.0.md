# Performance Benchmark — v3.1.0

> Added `api.slothlet.env` snapshot, `env.include` allowlist, reload immunity.  
> Benchmark date: April 14, 2026  
> Node.js runtime with `--conditions=slothlet-dev`

## Summary

| Metric | Eager | Lazy | Winner | Delta |
|--------|-------|------|--------|-------|
| **Startup** | 48.98ms | 35.14ms | Lazy | 1.4x faster |
| **Function Calls** | 13.78μs | 9.73μs | Lazy | 1.4x faster |
| **Materialization** | — | 1.19ms | — | one-time |

## Startup Time (10 iterations)

```
Eager Mode:  48.98ms (avg) | 34.10ms (min) | 144.83ms (max)
Lazy Mode:   35.14ms (avg) | 28.12ms (min) |  73.03ms (max)
```

## Function Call Performance (100 iterations)

```
Eager Calls:         13.78μs (avg) |  8.02μs (min) | 185.80μs (max)
Lazy Subsequent:      9.73μs (avg) |  6.49μs (min) |  70.20μs (max)
Lazy First Call:      1.19ms (materialization)
```

## Complex Module Access (100 iterations)


## Notes

- Env snapshot feature adds negligible overhead — startup and call times are within noise of v3.0.0.
- Lazy calls flip to being faster in Test 2 this run (13.78μs eager vs 9.73μs lazy) — V8 JIT variance at these microsecond scales.
