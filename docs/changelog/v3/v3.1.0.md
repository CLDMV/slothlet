# Slothlet v3.1.0 Changelog

**Release Date**: March 30, 2026
**Release Type**: Minor
**Branch**: `release/3.1.0`

---

## Overview

Version 3.1.0 adds `api.slothlet.env` — a frozen snapshot of `process.env` captured at initialization time and exposed through the protected `slothlet` namespace. This gives every module in a Slothlet instance a single, consistent, read-only view of the environment that was in effect when the instance was created, without relying on the mutable global `process.env` directly.

An optional `env.include` allowlist lets you restrict which keys are captured, keeping the snapshot minimal and auditable.

No breaking changes. All v3.0.x configuration and API usage is fully compatible.

---

## 🚀 New Features

### `api.slothlet.env` — Frozen Environment Snapshot

A frozen copy of `process.env` (or a filtered subset) is now available at `api.slothlet.env` after `load()` completes.

```javascript
const api = await slothlet({ dir: "./api" });

// Full snapshot — all keys present at initialization time
console.log(api.slothlet.env.NODE_ENV);   // "production"
console.log(api.slothlet.env.DATABASE_URL); // "postgres://..."

// The snapshot is frozen — mutations throw in strict mode
api.slothlet.env.NODE_ENV = "test"; // ❌ TypeError in strict mode, silent in sloppy
```

Inside a module, access the snapshot via `self.slothlet.env`:

```javascript
import { self } from "@cldmv/slothlet/runtime";

export function connectToDatabase() {
    const url = self.slothlet.env.DATABASE_URL;
    // ... use url
}
```

**Properties**:

| Property | Type | Description |
|----------|------|-------------|
| `api.slothlet.env` | `Readonly<Record<string, string \| undefined>>` | Frozen snapshot of `process.env` at `load()` time |

---

### `env.include` — Key Allowlist

Restrict the snapshot to a specific set of keys using the `env.include` option:

```javascript
const api = await slothlet({
    dir: "./api",
    env: {
        include: ["NODE_ENV", "DATABASE_URL", "LOG_LEVEL"]
    }
});

// Only the listed keys are captured
console.log(api.slothlet.env.NODE_ENV);    // "production"
console.log(api.slothlet.env.SECRET_KEY);  // undefined (not in include list)
```

**Allowlist behaviour rules**:

- `include` must be a non-empty array of strings to take effect.
- An empty array (`include: []`) is treated as no restriction — the full snapshot is captured.
- Non-string entries in the array are silently ignored.
- Keys listed in `include` that do not exist in `process.env` at initialization time are **not** included in the snapshot (no `undefined` placeholders).
- Any other value for `env` (e.g., `null`, a string, `true`) is ignored and the full snapshot is captured.

**Examples of include handling**:

```javascript
// All of these capture the full snapshot (no filtering):
env: undefined          // no config provided
env: {}                 // object but no include
env: { include: [] }    // empty array → treated as no restriction
env: { include: [42] }  // non-string entries only → treated as no restriction
env: "NODE_ENV"         // non-object value → ignored

// These capture only the listed string keys that exist in process.env:
env: { include: ["NODE_ENV", "PORT"] }
env: { include: ["NODE_ENV", 42, null, "PORT"] } // 42 and null are ignored
```

---

### Reload Immunity

The snapshot is captured **once** — on the first `load()` call — and is never replaced on subsequent `api.slothlet.reload()` calls. This ensures the snapshot always reflects the environment at instance creation time, not at reload time.

```javascript
process.env.FEATURE_FLAG = "enabled";
const api = await slothlet({ dir: "./api" });

// Snapshot captured here with FEATURE_FLAG = "enabled"
console.log(api.slothlet.env.FEATURE_FLAG); // "enabled"

// Mutate process.env after init
process.env.FEATURE_FLAG = "disabled";

// Reload does NOT refresh the snapshot
await api.slothlet.reload();
console.log(api.slothlet.env.FEATURE_FLAG); // "enabled" (unchanged)
```

## ⚙️ Configuration Reference

```javascript
const api = await slothlet({
    dir: "./api",

    // Optional — defaults to full snapshot if omitted
    env: {
        include: ["NODE_ENV", "PORT", "DATABASE_URL"]
    }
});
```

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `env` | `object \| undefined` | `undefined` | Top-level env config. Non-object values are ignored. |
| `env.include` | `string[]` | `undefined` | Allowlist of keys to capture. Empty array or non-array → full snapshot. |

## 🔄 Upgrade from v3.0.x

