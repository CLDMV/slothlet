# Slothlet v3.2.3 Changelog

**Release Date**: April 13, 2026
**Release Type**: Patch
**Branch**: `release/3.2.3`

---

## Overview

Version 3.2.3 is a patch release that fixes the publish workflow. The workflow was previously triggered by a pull request close event, which required the PR to be merged for the job condition to pass. Switching to a `push` trigger on `master`/`main` makes the publish run immediately when a squash-merged PR lands, without depending on the pull request event payload.

No source code, public API, or configuration changes. All v3.2.x usage is fully compatible.

---

## 🔧 CI / Infrastructure

### Publish Workflow: Switch Trigger to `push`

**Problem**: `publish.yml` was triggered on `pull_request: types: [closed]` and guarded by `github.event.pull_request.merged == true`. GitHub's squash-merge flow does not always fire a `pull_request.closed` event in time (or with the expected payload) when the merge is performed by the release automation, causing the publish job to be skipped.

**Fix**: Changed the trigger to `push: branches: [master, main]` so the workflow fires as soon as commits land on master. Added `paths-ignore` to skip doc-only pushes (`.md` files, issue/PR templates). Updated the job condition to `github.event_name == 'push' || github.event_name == 'workflow_dispatch'`.

**Permissions**: Bumped `contents` permission from `read` to `write` at the top-level and per-job so the workflow can create tags and GitHub Releases as part of the publish flow.
